PDF Remediation tool

The U.S. Department of Justice’s ADA Title II regulations require state and local governments to ensure that websites, mobile apps, and digital content meet WCAG 2.1 Level AA. Course readings and handouts published as PDFs are part of that digital ecosystem, and they are often harder for people using assistive technology to use than well-structured web pages. PDF Remediation helps by rebuilding selected PDFs as accessible HTML in Brightspace—structured text, headings, lists where appropriate, and alt text for figures—so you can offer a more accessible experience aligned with those expectations. Final conformance still benefits from your review, especially for complex tables and layout.

This guide explains how to use the tool: workflow, embed code, document types, and statuses. It is written for instructional designers who run the tool inside a course.

What the tool does

In the Title II / WCAG 2.1 AA context, moving high-value PDFs to semantic HTML is a practical way to improve perceivability, operability, and understandability compared to leaving the only copy as a flat PDF. The agent discovers PDF files already placed in your course Content (organized by module). For each PDF you choose, it can:

  • Assess the file quickly (page count and rough time/cost estimates)—no AI call for this step.
  • Remediate by transcribing each page, preserving figures, building structured HTML, and generating alt text for images.
  • Deploy a new Brightspace topic titled “(Original title) (Accessible HTML)” in the same module, pointing at the generated HTML in course files.
  • When the PDF included images, charts, or other figures, those visuals appear in the deployed HTML. Instructor-level roles can open that accessible topic in the course and use the built-in Crop tool on each figure to trim extra margins, template chrome, or unwanted background so learners see the part of the image that matters. Crops are saved as a separate image file next to the original extraction—the first version is not overwritten. While editing, you can usually switch between original and cropped previews; after a crop is saved, students typically see the cropped version by default.
  • Let you download a Word (.docx) version after remediation for editing or offline review.

Your original PDF topics are not removed automatically. Students can still open the PDF until you hide or replace it in Content if you choose.

Before you start

  • Add PDF Remediation.html to your course.
  • PDFs must live in the course Content structure so the tool can list them by module.
  • Large documents take longer; remediation runs one PDF at a time in a queue.

Workflow (step by step)

  1. Open PDF Remediation from the Content topic where you placed PDF Remediation.html (see Add PDF Remediation to your course).
  2. Choose the module from the dropdown to focus on PDFs in that unit.
  3. Use Select All Unremediated or check individual PDFs. Clear selection anytime.
  4. For PDFs not yet remediated: click Assess Selected. You’ll see page counts (and estimates). Per PDF, set Document type, optional Custom instructions, and checkboxes for Remediate and Deploy after.
  5. Click Start Remediation for the queued PDFs. Watch the progress screen until each run completes.
  6. After completion, use View Report for details. If you did not deploy during the run, use Deploy when you are ready to publish the HTML topic.
  7. If the source PDF had important images or figures, open the new (Accessible HTML) topic in Content. Use Crop on any figure that still shows extra slide background or margins, then save—so the preserved image is framed the way you want for the class.
  8. Optional: Download Word to edit in Microsoft Word; you may re-upload or adjust content manually if needed.
  9. For PDFs already remediated: Recompile HTML re-runs the pipeline (e.g., after prompt tweaks). Reset Selected removes deployed topics and optionally deletes generated course files—use with care (see below).

Scenario A — First-time remediation for a reading
  1. Select the module that contains the reading PDF.
  2. Check the PDF → Assess Selected.
  3. Choose document type General / Article (or a closer match).
  4. Leave Deploy after unchecked if you want to review HTML before publishing; otherwise check it.
  5. Start Remediation → when done, open the new “(Accessible HTML)” topic or deploy manually.
Scenario B — Instructor slide deck exported as PDF
  1. Assess as usual, then set document type to Slide Deck so bullets and slide structure are preserved.
  2. Add custom instructions only if something special must be emphasized (e.g., “Do not transcribe university logo watermark”).
  3. Run remediation; verify headings and lists in the HTML topic before hiding the old PDF if desired.
Scenario C — Undo a mistaken deploy
  1. Select remediated PDFs → Reset Selected.
  2. Remove from Content — removes only the deployed HTML topic; generated files stay in course files.
  3. Remove Everything — also deletes generated course files for those documents. Use when you need a clean slate.

Document types (pick the best match)

The document type tells the AI how to interpret layout. Hover options in the real app for short descriptions.

Type Best for
General / Article Articles, reports, essays, text-heavy PDFs.
Slide Deck Exported presentation slides—preserves titles, bullets, speaker notes when visible.
Annual Report / Magazine Rich layouts: sidebars, callouts, infographics as figures.
Brochure / Flyer Marketing-style multi-column layouts, CTAs, contact blocks.
Checklist / Worksheet Forms, blanks, checkboxes, numbered questions.
Scanned Document Book scans, photocopies—handles gutters and partial adjacent pages more carefully.
Custom instructions (optional). One or two sentences can steer emphasis (e.g., “Preserve all data tables verbatim” or “Skip decorative cover art”).

Statuses, deploy, and reset

Status badges

  • Not Remediated — no successful run yet (or reset removed the report).
  • In Process — remediation is running for that PDF.
  • Remediated — HTML (and supporting files) was generated; you can view the report, deploy if needed, or export Word.
  • Deployed — an accessible HTML topic exists in the module for that output.

Deploy

Deploy creates a new content topic; it does not overwrite the PDF topic. Titles append (Accessible HTML) so students can tell versions apart.

Reset options

  • Remove from Content — unpublishes the generated HTML topic from the module. Course files remain (you can deploy again).
  • Remove Everything — removes the HTML topic and deletes generated files for that document from course files. Use when you need to discard outputs entirely.
