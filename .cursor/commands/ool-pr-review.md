You are reviewing a **uga-ool** pull request for UGA Online.

Follow the **reviewer track** in skill `ool-pr-and-code-review` and `docs/cursor/pr-and-code-review.md`.

1. Ask for the PR URL or number if not provided. Use `gh pr view` and `gh pr diff` when available (read-only).
2. Run the team reviewer checklist: scope, secrets/FERPA, reference-repo edits, eLC test plan, monorepo `apps/` path, `VITE_*` client env, design system, agent framework imports.
3. Prefer **Ask mode** mindset — explain and comment; do not merge or push unless the user explicitly asks.
4. Deliver:
   - Brief summary of what the PR does
   - Blocking issues (must fix before merge)
   - Non-blocking suggestions
   - Draft text the reviewer can paste into GitHub review comments

Address the user's PR or follow-up questions.
