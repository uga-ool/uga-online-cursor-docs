From VS Code to Cursor — What Changes?

Cursor is built as a fork of VS Code. If you've been working in VS Code, you are already 90% of the way there. This section covers exactly what's the same, what's new, and what's genuinely better.

🔵
The One-Line Summary Cursor is VS Code with a deeply integrated AI co-pilot that understands your entire project — not just the file you have open. Your muscle memory, extensions, themes, and keybindings all carry over automatically.

What Stays Exactly the Same

⌨️

All Your Keybindings

Every VS Code shortcut works identically in Cursor. Ctrl+P, Ctrl+Shift+P, Ctrl+Z — no relearning required.

🧩

Your Extensions

Cursor supports the full VS Code Marketplace. Every extension you rely on installs and runs as normal.

🎨

Themes & Settings

Your color themes, font choices, and editor preferences transfer directly. Import your VS Code profile in one click during first setup.

📁

File Explorer & Layout

The sidebar, panel layout, and activity bar are identical. Everything is in exactly the same place.

The Key Mental Shift In VS Code, AI helps you write one line or function at a time. In Cursor, AI understands your entire project and can plan, edit, and execute across multiple files. Think of it as moving from "AI autocomplete" to "AI collaborator."

Bringing Your Projects Over from VS Code

Moving your existing work into Cursor is seamless — there is nothing to convert or export. Your project folders are identical in both editors.

Step 1 — Import Your VS Code Settings (First Launch)

When you first open Cursor, it offers to import your VS Code configuration automatically. This brings over your extensions, themes, keybindings, and preferences in one click.

📊 VS Code → Cursor Migration Flow

Step 2 — Open Your Existing Projects

  1. 1
    Just open the same folder Go to File → Open Folder and select your existing project directory. Nothing in the folder changes — Cursor reads it exactly as VS Code did.
  2. 2
    Cursor indexes your project automatically On first open, Cursor semantically indexes your files in the background, giving the AI full project awareness. A small indicator appears in the status bar; takes 1–2 minutes for large folders.
  3. 3
    Add a .cursorignore file for large projects (optional) Works like .gitignore — tells Cursor which files to exclude from AI indexing (large data exports, generated output). Keeps context clean and fast.
  4. 4
    Add team AI rules to the project folder Create a .cursor/rules/ folder and add Markdown files with your team's content standards. Commit this to version control so everyone gets the same AI behavior automatically.
💡
Pro Tip: Keep VS Code Around During Transition Both editors use the same files, so you can switch back and forth with no conflicts. Most team members find they stop opening VS Code within 1–2 weeks naturally.

Agent Modes — All Five Options

When you open the chat panel (⌘I / Ctrl+I) and click the mode dropdown, you'll see five options. Agent is the default. Each mode gives you a different level of AI autonomy and control.

∞ Agent (Default · ⌘I)

Open with ⌘I / Ctrl+I

What it does: Fully autonomous. Reads your project, plans a multi-step approach, edits multiple files, runs terminal commands, and iterates until the task is done.

Best for: Multi-file tasks, QA sweeps, generating or restructuring content at scale. This is your workhorse mode.

Think of it as: Sending a capable colleague off to work on a well-defined task. Review their output before treating it as final.

⚡ Autonomous · Always review output

≔ Plan

Select from mode dropdown

What it does: Agent researches your project, asks clarifying questions, and generates a reviewable Markdown plan before writing any code or making any edits.

Best for: Any large or risky task where you want to approve the approach before execution. The plan file is editable — remove steps, adjust scope, then say "go ahead."

Think of it as: Agent Mode with a mandatory checkpoint before anything changes.

📋 Plan first · Then approve

🐛 Debug

Select from mode dropdown

What it does: Instruments your project with runtime logs, helps reproduce problems, and tracks down the root cause of bugs across stacks and languages.

Best for: Tracking down why a script, data pipeline, or web page isn't behaving as expected. Works by observing actual runtime behavior, not just reading code.

Think of it as: A debugging partner that instruments your code and interprets the results.

🔬 Runtime diagnostics

◎ Multitask

Select from mode dropdown

What it does: Runs multiple agent tasks in parallel, each in its own isolated workspace. Useful when you have several independent tasks that don't affect each other.

Best for: Processing several modules simultaneously, running parallel QA checks on independent files, or splitting a large batch job into concurrent streams.

Think of it as: Assigning the same job to several agents at once and comparing results.

⚡ Parallel execution

💬 Ask

Select from mode dropdown  |  Read-only

What it does: Read-only AI conversation. Ask questions, get explanations, explore options. The AI never edits any files in this mode — it can only read and respond.

Best for: Understanding a file, planning a task, getting feedback before committing to a change.

Think of it as: A smart colleague you can ask anything — without worrying they'll accidentally change something.

Pro tip: Use Ask first to scope and plan, then switch to Agent to execute. This combination produces the best results.

🔒 Read-only · Safe to explore · Great starting point
📊 Which Mode Should I Use?

The Power Workflow: Ask → Plan → Agent → Manual Inline Edit

🔁
Recommended Pattern for Most Tasks
  1. Ask mode first — Scope the task. "What are the accessibility gaps in this module?" Review the AI's analysis without risking any changes.
  2. Plan mode for complex work — For large tasks, switch to Plan. Review and edit the Markdown plan before anything executes.
  3. Agent mode to execute — "Now go ahead and fix those issues across all five modules." Review the diff output when it finishes.
  4. Inline edit to polish — Select any sentence or paragraph, press ⌘K / Ctrl+K, and make a precise final tweak.
✏️
Inline Editing — the Manual Edit Shortcut You don't need to open the chat panel for small targeted changes. Select any text in the editor and press ⌘K (Mac) or Ctrl+K (Windows). A small inline prompt bar appears — describe your change and the AI proposes a diff right in place. No mode switching needed. This is the fastest way to rewrite a single sentence or fix a specific line.

Using Plan Mode in Agent

In Agent Mode, ask for a reviewable plan before any changes are made. This is especially valuable for large or complex tasks:

# Instead of "Go fix all the modules", try: "Before making any changes, create a plan for how you would audit all 5 course modules for Bloom's Taxonomy alignment. List the steps and what you'll check in each file. Wait for my approval." # Agent creates a Markdown plan file you can edit directly. # Remove steps you don't need, then say: "Looks good — go ahead."

AI Chat & the @ Reference System

The @ symbol is your most important tool in Cursor. It tells the AI exactly what context to look at, which keeps responses focused and reduces unnecessary token use.

@ References — What You Can Point To

@ CommandWhat It DoesExample Use
@filenameInclude a specific file in AI context@module1.html summarize the learning objectives
@folderInclude all files in a folder@modules/ are all five consistent in tone?
@codebaseSearch entire indexed project@codebase where do we define accessibility standards?
@webSearch the web for current information@web WCAG 2.2 guidelines for e-learning 2026
@BranchContext from your current git changes@Branch what have I changed in this session?
@cursor-rulesReference your team's AI rules@cursor-rules what are our content standards?

Sample Prompts for Your Teams

INSTRUCTIONAL DESIGN — BLOOM'S AUDIT

"@module3.html Review each learning objective and tell me which Bloom's Taxonomy level it targets. Flag any that aren't measurable. Suggest one rewrite for each flagged objective."

QUALITY REVIEW — HTML SWEEP

"@modules/ Scan all files for broken image tags, missing alt text, unclosed HTML tags, and inconsistent heading levels. Return a numbered list with filename and line number for each issue."

ANALYTICS — QUIZ DATA

"@quiz-results.csv Calculate mean score, median, and pass rate (≥70). Identify the 3 questions with the lowest average correct response rate. Format as a summary table."

CROSS-MODULE CONSISTENCY

"@module1.html @module2.html @module3.html Are these three modules consistent in reading level, tone, and structure? Flag the top 3 inconsistencies and suggest how to address each."

🔒
Privacy Note — FERPA Do not include student names, UGA IDs, or other individually identifiable student data in AI prompts. Aggregated, anonymized data (quiz averages, group pass rates) is generally fine. When in doubt, remove identifying fields from your file before opening it in Cursor.

The Context Wheel — What "100% Context Used" Means

In the chat input bar
100% context used 🖼 🎤
The circle fills as context fills up

The small circle icon in the chat input bar is a context usage indicator. It fills as you add more files, references, and conversation history to a single chat session.

  • Circle mostly empty — Plenty of context remaining. You can continue adding @references and messages.
  • Circle mostly full — Context is getting tight. AI responses may become less precise or start ignoring earlier parts of the conversation.
  • "100% context used" — The context window is full. Start a new chat. The AI cannot reliably use more information in this session. Type /summarize first if you want to carry key points forward.
⚠️
When you see 100% context used — start a fresh chat Don't keep pushing messages into a full context. The AI will silently start ignoring the oldest parts of the conversation, producing inconsistent or contradictory results. Open a new chat, include only the files you still need, and continue from there.

Tips, Tricks & Gotchas

✅ Best Practices

  1. ✔
    Plan in Ask, execute in Agent Don't jump straight to Agent for complex tasks. Spend two minutes in Ask mode scoping the work — it produces cleaner results and avoids wasted credits on misdirected runs.
  2. ✔
    Always review AI edits before accepting Cursor shows a diff. Read it. AI can produce confident errors. A 30-second review catches 90% of issues before anything is saved.
  3. ✔
    Commit your .cursor/ folder to version control This is how the whole team shares rules and commands. Treat it like any other config file — it belongs in Git alongside your project files.
  4. ✔
    Start fresh chats for unrelated tasks Long chats accumulate noise that can cause the agent to lose focus. New task = new chat. Use /summarize if you need to carry context forward from a long session.
  5. ✔
    Stay on Auto mode by default Auto mode is effectively unlimited and Cursor selects the appropriate model per task. Only manually switch to Opus when you have a genuine need for its extra power on a complex task.

⚠️ Common Gotchas

⚠️
AI Can Be Wrong — Confidently If the AI reports "no issues found" or "all changes complete," verify it. AI models can miss things or introduce subtle new problems. Human review remains essential, especially for content going to students.
⚠️
Long Agent Sessions Degrade Quality After many back-and-forth turns, agents can lose focus and start contradicting earlier work. For long projects, break the task into chunks and start a new chat session for each one.
⚠️
Watch the Context Wheel — "100% Context Used" The circle icon in the chat input bar fills as your session accumulates files, references, and messages. When you see "100% context used", start a new chat immediately. The AI cannot reliably use additional information and may start silently ignoring earlier parts of the conversation, producing inconsistent results.
